# 解决方案与角色

## 什么是解决方案

解决方案是一组预配置好的智能体能力集合。通过解决方案，可以为智能体指定特定的技能和角色，让它可控地解决特定领域的问题。

**举例：**

- 「软件工程」解决方案：包含需求分析、UI 设计、任务拆分等角色，覆盖软件开发的各个环节。
- 「智能招投标」解决方案：包含标书分析、合规检查等角色，每个角色只使用招投标相关技能，确保输出专业、准确。

## 选择和切换解决方案

打开「智能助手」后，在顶部选择你需要的解决方案。不同解决方案提供不同的能力和角色组合。

```
+------------------------------------------------------------------+
|  智能助手                                                         |
|------------------------------------------------------------------|
|  解决方案: [软件工程 v]   角色: [需求分析 v]                       |
|                          ← 点击下拉切换解决方案或角色               |
|------------------------------------------------------------------|
|                                                                  |
|  (对话内容区域)                                                   |
|                                                                  |
+------------------------------------------------------------------+
```

## 角色的作用

同一个解决方案下可以有多个角色，每个角色擅长处理不同类型的任务。例如在「软件工程」解决方案中：

- 「需求分析」角色专注于梳理和分析需求
- 「UI 设计」角色专注于界面设计
- 「任务拆分」角色专注于将需求拆解为可执行的开发任务

选择合适的角色，能让智能体更高效地完成你的需求。

## 切换角色

切换角色时会自动创建新的会话。这是因为不同角色使用不同的技能和上下文，新会话可以确保智能体以正确的身份和能力来回答你的问题。

## 解决方案应用（Solution App）

某些解决方案可以作为独立应用运行，拥有定制化的 UI 框架。例如「智能标书」应用，提供了专门的标书编辑和审查界面，而不是通用的对话窗口。

这类应用会出现在 Blade OS 桌面的应用列表中，点击即可打开。
